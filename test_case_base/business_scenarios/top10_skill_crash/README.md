# Top 10 Skills 高并发崩溃测试

## 概述

本测试用例用于**高并发、高频度**调用 OpenClaw 最常用的 **Top 10 插件**，监测系统资源（CPU、内存）的变化，找出服务器崩溃的临界点。

## 测试目标

- **并发用户数**: 10 → 50 → 100 → 200 → 500 逐步递增
- **总调用次数**: 数万至数十万级
- **监测指标**: CPU%、内存%、响应时间、并发连接数
- **终止条件**: 系统达到崩溃临界点（内存≥95% / CPU≥98% / 可用内存<100MB）

## Top 10 测试插件

| 排名 | 插件名 | 权重 | 复杂度 | 模拟操作 |
|------|--------|------|--------|----------|
| 1 | web-tools-guide | 25% | high | Web搜索/抓取决策 |
| 2 | browser | 20% | high | 浏览器自动化 |
| 3 | github | 15% | medium | GitHub API调用 |
| 4 | file-ops | 10% | low | 文件读写操作 |
| 5 | memory | 8% | medium | 内存操作 |
| 6 | sessions | 7% | medium | 会话管理 |
| 7 | message | 5% | low | 消息发送 |
| 8 | github-analysis | 4% | high | GitHub数据分析 |
| 9 | search | 3% | medium | 搜索工具 |
| 10 | weather | 3% | low | 天气查询 |

## 负载阶段

测试分5个阶段逐步加压：

| 阶段 | 并发数 | 持续时间 | 描述 |
|------|--------|----------|------|
| 1 | 10 | 30s | 轻度负载 - 基准测试 |
| 2 | 50 | 60s | 中度负载 - 正常压力 |
| 3 | 100 | 120s | 重度负载 - 高压力 |
| 4 | 200 | 300s | 极限负载 - 压力测试 |
| 5 | 500 | 600s | 崩溃测试 - 寻找临界点 |

## 使用方法

### 直接运行

```bash
python top10_skill_crash_test.py
```

### 带配置运行

```bash
python top10_skill_crash_test.py '{"crash_memory_threshold": 90}'
```

### 指定输出文件

```bash
python top10_skill_crash_test.py '{}' top10_crash_report.json
```

## 监测指标

### 实时输出
```
[运行  45.2s] 调用:  5234 | 并发:  50 | CPU:  45.3% | 内存:  67.8% | 已用:   3456MB
```

### 崩溃检测
当达到以下条件时测试自动终止：
- 内存使用 ≥ 95%
- CPU 使用 ≥ 98%
- 可用内存 < 100MB

## 测试报告

测试完成后生成 JSON 报告，包含：

- **执行统计**: 总调用次数、成功率、平均响应时间
- **资源峰值**: CPU%、内存%、最大并发数
- **阶段结果**: 每阶段的性能数据
- **技能统计**: 每个插件的调用分布
- **崩溃信息**: 崩溃原因、发生阶段
- **完整时间线**: 所有采样点的详细数据

## 预期结果

### 正常情况
- 完成所有5个阶段测试
- 系统在500并发下仍稳定运行
- 收集到完整的性能基线数据

### 崩溃情况
- 在某个阶段触发崩溃条件
- 记录崩溃时的临界值
- 为系统优化提供数据支持

## 注意事项

⚠️ **警告**: 本测试会消耗大量系统资源，建议：
1. 在测试/开发环境运行
2. 确保服务器有足够的内存（建议8GB+）
3. 不要在生产环境运行
4. 测试期间避免其他重要任务
