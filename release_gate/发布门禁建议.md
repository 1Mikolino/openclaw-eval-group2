##  发布门禁建议

### 阻断项

出现任一项，禁止发布：

- Gateway 无法启动或日志未出现 `gateway ready`。
- 默认模型为 `mock-blackhole/blackhole-model`。
- 日志出现 `Unknown model`、`Unauthorized`、`401`、`403` 并影响主流程。
- Smoke/白盒矩阵失败，即 `web_tools_guide_whitebox_matrix.py` 返回非 0。
- Ontology 真实白箱矩阵失败，即 `ontology_real_whitebox_test.py` 返回非 0。
- 多浏览器并发 5/10/20 任一档出现崩溃。
- `issue-97741-result.txt` 为 FAIL。
- `issue-97655-result.txt` 为 FAIL。
- Top 10 Skill 轻量压测出现非预期 crash。
- 多 Agent 或渠道路由出现 session 串线、错误 agent 身份、静默 fallback。

### 性能门禁

多浏览器并发：

- 5、10、20 并发均需完成。
- `summary.success_rate` 必须为 `100%`。
- `resource_usage.peak_memory_percent` 不得超过 `95%`。
- `resource_usage.peak_cpu_percent` 不得超过 `95%`。

Skill 稳定性：

- `summary.success_rate >= 0.95`。
- `summary.avg_response_time_sec <= 1.0`，如脚本模拟环境与真实环境差异较大，可放宽到 2 秒。
- `resource_usage.memory_growth_mb <= 100`。
- `events` 为空；如非空，必须人工确认不是 warning 级误报。

Ontology 真实白箱矩阵：

- `summary.success_rate == 1.0`。
- `summary.failed == 0`。
- `judge_contract.entity_extraction_in_test_script == false`。
- 任一 Strict Equal 路径缺失、实体名顺序不一致、类型契约不一致，均视为失败。

Top 10 Skill 崩溃边界：

- 轻量档不得出现 crash。
- 完整档允许找到极限边界，但必须生成完整报告。
- 如果在 50 并发以内 crash，禁止发布。

###  回归对比门禁

候选版本相对基线版本：

- P90 任务耗时增长不得超过 `25%`。
- P95 任务耗时增长不得超过 `35%`。
- P90 Token 消耗增长不得超过 `30%`。
- 工具调用错误率不得上升超过 `5%`。
- 心跳、空转、重试 Token 不得异常增长。

### 稳定性门禁

- 5 分钟资源采样中 RSS 不得持续单调增长。
- 无僵尸进程堆积。
- 无浏览器/sidecar 进程无法回收。
- 无频繁 `liveness warning`、`event_loop_delay`、`queue age` 持续增长。
- 无 `AbortError` 被错误归类为模型 timeout。
